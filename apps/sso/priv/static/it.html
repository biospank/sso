    Introduzione

    Descrizione

    SSO è un sistema di autenticazione centralizzato per web e mobile. Espone servizi (endpoints) in modalità REST per la registrazione, autenticazione e notifiche mail.

    Gli account registrati che intendono aderire al sistema di autenticazione hanno a disposizione le seguenti funzionalità.

    • Sessione account
    • Registrazione utente (double opt-in)
    • Autenticazione utente
    • Reset password utente (double opt-in)
    • Cambio password utente
    • Dettaglio utente
    • Aggiornamento profilo utente

    Sessione

    Descrizione

    SSO usa chiavi di accesso per l'utilizzo dele API. Per richiedere le chiavi di accesso contattare il provider Dardy.

    Esempio

    
# cURL

curl -i
  -X POST
  -d '{"account": {
        "access_key": "<access_key>",
        "secret_key": "<secret_key>"
      }
    }'
  -H "Accept: application/vnd.dardy.sso.v1+json"
  -H "Content-Type: application/json"
  https://dardy-sso-test.herokuapp.com/sso/session

    # Ruby

require 'httparty'

HTTParty.post('https://dardy-sso-test.herokuapp.com/sso/session',
  body: '{"account": {
            "access_key": "<access_key>",
            "secret_key": "<secret_key>"
           }
         }',
  headers: {
    'Accept' => 'application/vnd.dardy.sso.v1+json',
    'Content-Type' => 'application/json'
  }
)

    // jQuery

$.ajax({
  type: "POST",
  url: "https://dardy-sso-test.herokuapp.com/sso/session",
  data: '{' +
    '"account": {' +
      '"access_key": "<access_key>",' +
      '"secret_key": "<secret_key>"' +
    '}' +
  '}',
  headers: {
    "Accept": "application/vnd.dardy.sso.v1+json",
    "Content-Type": "application/json"
  }
});

    Ritorna un JSON strutturato come segue:

    {
  "account":
  {
    "id":2,
    "app_name": "Test",
    "active": true,
    "access_key": "<access_key>"
  }
}

    Richiesta HTTP

    POST https://dardy-sso-test.herokuapp.com/sso/session

    Headers HTTP

    Accept: application/vnd.dardy.sso.v1+json

    Sostituire <access_key> e <secret_key> con le chiavi di accesso fornite da Dardy.

    Headers risposta HTTP

    authorization: Dardy <jwt>

    <jwt> è il token che deve essere utilizzato per le chiamate agli altri endpoints.

    x-expires: <timestamp>

    <timestamp> indica la data di scadenza della sessione (1 giorno)

    Codici risposta HTTP

    Codice Descrizione
    201 Created – La richiesta è andata a buon fine
    404 Not Found – Account non trovato
    401 Unauthorized – Account non autorizzato
    422 Unprocessable Entity – Errore di validazione
    423 Locked – Account temporaneamente disabilitato

    Registrazione Utente

    Descrizione

    Sono possibili due tipi di registrazione:

    1. Registrazione con attivazione
    2. Registrazione preautorizzata

    La registrazione con attivazione si completa in due fasi:

    • Registazione
    • Attivazione

    La registrazione preautorizzata si completa in una fase:

    • Registazione

    Registazione con attivazione

    Descrizione

    Crea un utente non attivo in stato non verificato. Invia una mail all'indirizzo specificato dall'utente per la procedura di registrazione e una notifica via mail all'indirizzo dell'account.

    Esistono due template per la mail di registrazione:

    1. Con codice attivazione (mobile)
      • Escludendo callback_url dal corpo del messaggio HTTP
    2. Con link alla pagina di attivazione (web)
      • Includendo callback_url nel corpo del messaggio HTTP
      • il codice attivazione viene appeso all'url callback_url

    Privacy

    Il consenso per la privacy è obbligatorio e viene dato sia all'app/sito da cui si effettua la registrazione dell'utente sia al servizio di SSO.

    I consensi obbligatori sono:

    1. privacy_consent: true
    2. sso_privacy_consent: true

    Se la registrazione viene completata correttamente, l'informazione della privacy data all'app/sito viene aggiunta nel campo:

    app_consents: [...]

    Consultare questo campo per verificare se un utente ha dato il consenso alla privacy per un determinato app/sito.

    Registazione preautorizzata

    Descrizione

    Crea un utente attivo in stato verificato. Invia una mail di cortesia all'indirizzo specificato dall'utente.

    Cambio Mail Utente

    Descrizione

    Il cambio della mail si completa in due fasi:

    • Richiesta cambio mail
    • Cambio mail

    Richiesta cambio mail

    Descrizione

    Invia una mail al nuovo indirizzo dell'utente che ha richiesto il cambio della mail.

    Esistono due template email per la richiesta di cambio mail:

    1. Con codice cambio mail (mobile)
      • Escludendo callback_url dal corpo del messaggio HTTP
    2. Con link alla pagina di cambio mail (web)
      • Includendo callback_url nel corpo del messaggio HTTP
      • il codice cambio mail viene appeso all'url callback_url

    Esempio

    
# cURL

curl
  -X POST

    Aggiornamento Profilo Utente

    Descrizione

    Aggiorna il profilo utente con i dati inviati nel corpo del messaggio HTTP.

    Privacy

    Il consenso alla privacy è obbligatorio e deve essere fornito dall'app/sito che effettua la registrazione dell'utente. Tale consenso può essere dato in fase successiva alla registrazione solo se il consenso proviene da un app/sito diverso da quello di registrazione.

    Il consenso da fornire è:

    1. privacy_consent: true

    Se l'aggiornamento del profilo viene completato correttamente, l'informazione della privacy data all'app/sito viene aggiunto nel campo:

    app_consents: [...]

    Esempio

    
# cURL

curl
  -X PUT
  -d '{"profile": {
        "phone_number": "nuovo numero di telefono",
        "profession": "nuova professione"
      }

    Codici Errore HTTP

    Descrizione

    SSO usa i seguenti codici di errore, oltre a quelli specifici, su tutti gli endpoints:

    Codice Descrizione
    406 Not Acceptable – Accept header non valido
    498 Authentication required – Token non valido
    500 Internal Server Error – Problemi sul server SSO.
    503 Service Unavailable – Offline per manutenzione. Riprovare più tardi.
